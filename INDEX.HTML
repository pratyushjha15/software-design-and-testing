Short Answer Conceptual

2‚Äì3 line answers focusing on core reasoning and design intuition.

SA 1
3 marks

1. Define a unit test. What are the three key properties it should have?

Sample answer ‚ñæ
A unit test is an automated test that verifies a single unit of behavior quickly and in isolation. It should be: fast, deterministic (repeatable), and independent of other tests.
SA 2
3 marks

2. Why is testability considered a design smell indicator? Give one example.

Sample answer ‚ñæ
When code is difficult to test (requires lots of setup or hidden state), it often reveals excessive coupling or violation of SRP. For example, a class that talks directly to a DB, sends emails, and implements business logic in the same method is both hard to test and poorly designed.
SA 3
3 marks

3. Distinguish between a mock and a stub.

Sample answer ‚ñæ
A stub provides canned responses for inputs but is not checked for how it was called; a mock is pre-programmed with expectations and the test fails if calls don‚Äôt match the expected interactions.
SA 4
3 marks

4. What is the main goal of refactoring? How does a good test suite support it?

Sample answer ‚ñæ
Refactoring aims to improve internal structure without changing observable behavior. A good test suite guards behavior, so you can confidently change internals, knowing you have a safety net against regressions.

Scenario 1

1. You have a class InvoiceFilter that connects to a real database, filters low-value invoices (< 100), prints them to console, and sends a notification. All of this happens in a single method.

Coding / GTest Tasks Implementation

Hands-on questions for writing tests and thinking about complexity.

Code 1
4‚Äì6 marks

1. Write Google Tests (using GTest) for a function int divide(int a, int b) with this behavior:

  • For b != 0, returns integer division a / b.
  • For b == 0, throws std::invalid_argument.

Include at least one normal case and one exception case.

// Example structure (answer outline)
TEST(DivideTest, NormalDivision) {
    // Arrange
    int a = 10, b = 2;
    // Act
    int result = divide(a, b);
    // Assert
    EXPECT_EQ(result, 5);
}

TEST(DivideTest, ThrowsOnZeroDivisor) {
    // Arrange
    int a = 10, b = 0;
    // Act + Assert
    EXPECT_THROW(divide(a, b), std::invalid_argument);
}
Code 2
4‚Äì6 marks

2. Implement a parameterized Google Test for a function bool isEven(int n) using the following (input, expected) pairs: (-2, true), (-1, false), (0, true), (1, false), (100, true). 

// Example structure (answer outline)
class IsEvenTest
    : public ::testing::TestWithParam<std::pair<int, bool>> {};

TEST_P(IsEvenTest, WorksForVariousInputs) {
    auto [input, expected] = GetParam();
    EXPECT_EQ(isEven(input), expected);
}

INSTANTIATE_TEST_SUITE_P(
    IsEvenValues,
    IsEvenTest,
    ::testing::Values(
        std::make_pair(-2, true),
        std::make_pair(-1, false),
        std::make_pair(0, true),
        std::make_pair(1, false),
        std::make_pair(100, true)
));
Code 3
4‚Äì6 marks

3. For the following nested loop, determine the time complexity and briefly justify: 

for (int i = 1; i < n; i *= 2) {
    for (int j = 0; j < n; ++j) {
        // constant work
    }
}
Sample answer ‚ñæ
Outer loop runs O(log n) times (i doubles each time), inner loop runs O(n) each time ‚áí overall O(n log n).
